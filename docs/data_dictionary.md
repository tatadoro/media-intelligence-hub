# Data Dictionary

Источник истины по формату данных:
- `contracts/silver.schema.json`
- `contracts/gold.schema.json`

Ниже — человекочитаемое описание ключевых полей.

## Silver layer (`data/silver/*_clean.json`)

- `id` — идентификатор материала в пайплайне (строка или число).
- `title` — заголовок.
- `link` — ссылка на источник.
- `source` — источник/домен (например, `lenta.ru`).
- `raw_text` — исходный текст (может содержать шум/обвязку).
- `clean_text` — очищенный текст, подготовленный для дальнейшей обработки.
- `body_status` — статус извлечения/обогащения body (например, успешно/нет/частично).
- `body_len` — длина body/текста в символах (или 0, если текста нет).

## Gold layer (`data/gold/*_processed.parquet`)

Поля исходного контента (приходят из silver):
- `id`, `title`, `link`, `source`, `raw_text`, `body_status`, `body_len`, `clean_text`

NLP-поля (обогащение):
- `nlp_text` — текст, который реально уходит в NLP (нормализованный/очищенный вариант для суммаризации и ключевых слов).
- `summary` — краткое резюме материала.
- `keywords` — ключевые слова (строка; формат зависит от реализации, например “через запятую”).

Текстовые метрики:
- `text_length_chars` — длина `nlp_text` (символы).
- `num_sentences` — число предложений (по эвристике пайплайна).
- `num_keywords` — число ключевых слов (по эвристике пайплайна).

Техническое поле батча:
- `ingest_object_name` — имя объекта/батча. В gold должно совпадать с именем parquet-файла и используется для дедупа/аудита загрузки в ClickHouse.
